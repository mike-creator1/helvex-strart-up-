Capabilities

What Nexus does well.

Four capabilities matter most when you're picking Nexus on purpose.

Extended thinking

Nexus reasons out loud when the question calls for it. Visible thinking traces let you see the path, not just the conclusion.

Citations engine

Every claim grounded in retrieved sources can be cited back to the line. Click a citation, see the source. This is on by default for retrieval calls.

Long context

Drop in entire briefs, transcripts, document bundles. Nexus holds them all in working memory for the whole exchange.

Parallel tools

Tool calls that don't depend on each other run in parallel. Multi-tool plans complete in one round-trip instead of N.

Technical posture

How Nexus behaves in production.

The operational posture you can plan around when you wire Nexus into a real system.

  • Thinking Extended (visible). Reasoning traces stream alongside the answer. Toggle off when you want pure output.
  • Citations HelveX Citations engine. Retrieved sources can be cited back to the line. Standard for retrieval calls.
  • Context Longest in the family. Holds whole-document and multi-document workloads without splitting.
  • Inputs Text and image. Vision input accepted alongside text and tool definitions.
  • Tool use Parallel · structured. Strict JSON schemas. Multiple independent tools dispatched in parallel.
  • Best paired with Ether for front-line triage, Prometheus for action-bound tasks that need a deeper plan.
