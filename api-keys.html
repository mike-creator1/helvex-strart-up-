How keys work

Generation

Server-side, never logged

Keys are created with cryptographically-random bytes inside Postgres. The plaintext is shown to you once and immediately hashed.

Scope

Workspace or project-bound

Bind a key to a single project, or leave it workspace-wide. Revoke a key without affecting the rest of your workspace.

Audit

Every issuance & revoke logged

Each create and revoke writes to Activity with timestamp and the key's last-four.
