Best practices

What we have learned from teams that ship.

The patterns that show up in every team that gets real value out of HelveX. Most are about workflow, not prompting.

  1. 01

    Pick the model on purpose.

    The router is good but it isn't psychic. If a task is latency-bound, depth-bound, or action-bound, pin it. The model picker takes 30 seconds to read.

  2. 02

    Use Projects for repeated work.

    Pin the documents and prompts every conversation should start from. The model has the context, the team has a shared starting point, and you stop re-pasting the same brief.

  3. 03

    Treat citations as the answer.

    For anything you will act on, click the citation. Nexus links every claim back to the line. If a claim is uncited, raise the bar before you trust it.

  4. 04

    Put humans on irreversible steps.

    Approval gates exist for sends, file writes, payments, and external tool calls. Default them on for shared Spaces, opt-out only when the cost of a wrong action is genuinely small.

  5. 05

    Replay before you blame.

    When an agent run goes wrong, the trace tells you where. Don't re-prompt — replay from the step that broke and edit the plan. You will fix the same bug in half the time.
