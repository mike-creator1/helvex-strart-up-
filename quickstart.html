What you'll have in 30 minutes

Three working surfaces.

Not toys, not demos — three surfaces wired into your workspace, ready to extend.

Chat

A pinned Chat thread

A persistent thread pinned to a chosen model — Ether, Nexus, or Prometheus. System prompt set, prompt caching warmed, history kept.

Project

A Project with seeded context

A scoped workspace with a handful of documents seeded into Knowledge. Retrieval-grounded answers, citations on every response.

Agent

An Agent with one approval gate

A single-step agent — read a queue, draft a reply, wait for human approval. Replayable trace from the first run.

Common first integrations

Where workspaces wire in first.

Six connectors most teams hook up in the first week. Standard OAuth or service-account flows; nothing custom required.

Messaging

Slack

Trigger agents from a channel, deliver drafts as DMs, route approvals into a private thread.

Docs

Notion

Index pages into Knowledge; ground answers in your wiki with traceable citations back to the source page.

Files

Drive

Pull docs, sheets, and PDFs into Knowledge. Permissions inherited; nothing crosses the boundary you set.

Code

GitHub

PR triage, issue summaries, code-review assists. Read-only by default; write access through an explicit approval gate.

Issues

Linear

Triage incoming issues, group duplicates, draft replies. Tasks open back into Linear with full context.

Identity

SSO

SAML or OIDC, SCIM provisioning. Workspace identity comes from your IdP; no extra logins for new joiners.

Tutorial format every page follows

Four steps, on every page.

Every tutorial in the library follows the same four-step arc. Predictable shape, no guessing what the page wants.

  1. 01

    Problem

    A single paragraph on the real-world problem. No abstractions; the tutorial is for someone who has the problem right now.

  2. 02

    Model choice

    Which model the tutorial uses and why — latency, depth, or action. A one-line pointer to the model picker for the routing rules.

  3. 03

    Workflow

    The step-by-step build. Screenshots, prompts, tool definitions. Replayable from a clean workspace.

  4. 04

    Debug

    What goes wrong the first time, and how to diagnose it. The failure modes you'll actually hit, written down once.
