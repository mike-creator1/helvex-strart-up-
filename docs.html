Authentication

Every AI endpoint expects a Supabase access token in the Authorization header. Tokens come from the browser session created during sign-in; you never construct them manually.

Authorization: Bearer <supabase_access_token>

The token is short-lived and refreshes automatically in the browser. Server-side integrations should use a HelveX API key (hxk_live_…) issued from API Keys; the same header shape applies.

Errors

Failure responses are always JSON with at least error and, when relevant, trace_id. The trace ID is the single string Support needs to investigate.

StatusMeaning
400Malformed body or a validation failure. The error string describes what's missing.
401Missing or invalid session token. Sign in again.
402Out of HelveX credits. Top up in Credits.
405Wrong HTTP method — every AI endpoint is POST.
500Server-side failure. The trace ID is the right thing to forward to Support.
502Upstream provider unreachable. Retry after a short backoff.
503A dependency (Stripe, Supabase) is not configured for the workspace.

Streaming format

Every AI endpoint returns Server-Sent Events. Each event line begins with event: or data:, separated by blank lines. The shapes mirror the upstream provider so a single SSE parser handles every endpoint identically.

event: message_start
data: {"type":"message_start","message":{"model":"nexus-4-5", ...}}

event: content_block_delta
data: {"type":"content_block_delta","delta":{"type":"text_delta","text":"Hello"}}

event: message_stop
data: {"type":"message_stop"}

The model field is sanitised server-side. Clients see HelveX brand names (nexus-4-5 / ether-3 / prometheus-4-5), never upstream model IDs.

AI endpoints

Each endpoint debits one HelveX credit before forwarding the request to the upstream model. New accounts ship with 10 000 starter credits.

crm-enrich

POST/api/crm-enrichBearer · 1 credit

Returns a structured sales-intelligence JSON for a contact (company guess, industry, size band, seniority, conversation starter, outreach angle, warnings) — every field carries an honest confidence level.

curl -X POST https://createx-gilt.vercel.app/api/crm-enrich \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"email":"maria@acme.de","name":"Maria Kotsoni","notes":"replied to LinkedIn ad"}'

generate-social

POST/api/generate-socialBearer · 1 credit

Three social-post variants (Instagram / LinkedIn / X / Facebook) from a one-line brief — sized, hashtagged, in your brand voice.

{ "brief": "Launch announcement for our new pricing tier",
  "platforms": ["linkedin","x"], "tone": "Professional but warm" }

generate-ads

POST/api/generate-adsBearer · 1 credit

Ad-copy variants for Meta, Google, or TikTok — headlines, body, CTAs, A/B variants — strictly within each platform's character budget.

generate-campaign

POST/api/generate-campaignBearer · 1 credit

Email-campaign drafts. Brief in, n finished drafts out (subject + preview + body), all in the requested tone, language, and audience.

automation-reply

POST/api/automation-replyBearer · 1 credit

Pastes in an incoming email; returns a classified triage object (intent, urgency, sentiment) plus a draft reply ready to send.

assistant-chat

POST/api/assistant-chatBearer · 1 credit

Streaming conversation with the HelveX Business Assistant. Persistent personas, multilingual replies, opinionated by default.

{ "message": "Draft a 1-page mutual NDA, Swiss law, 2-year term.",
  "history": [/* prior {role, content} pairs */],
  "system":  "/* optional persona system prompt */" }

builder-chat

POST/api/builder-chatBearer · 1 credit

The chat backend for every builder surface (agents, app, model, website, workflow). Accepts the same body shape as /api/assistant-chat plus a model brand name (ether, nexus, prometheus) that resolves to a real upstream model server-side.

Creates a Stripe Checkout Session for either a one-time consulting engagement or a recurring HelveX subscription. The slug decides which. Server-side price enforcement — the client never controls the amount.

Health

GET/api/healthPublic

Pings every critical dependency (Supabase auth, upstream AI, the function's own cold path) in parallel and returns structured JSON with per-service latency. Cache-Control: no-store so external monitors can poll every 30 s without staleness.

  "ms": 240,

  "services": [

    { "name": "supabase",  "ok": true, "ms": 120 },

}

Telemetry

POST/api/log-errorOptional Bearer

Browser error sink. window.onerror and unhandled promise rejections are forwarded here automatically by platform.js. Signed-in errors land in public.log_events tagged with the user's owner_id; anonymous ones drop silently.

The endpoint is rate-limited client-side (6 reports per page-load) and never blocks the page that produced the error.
