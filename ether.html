Ether Real-time. Lowest cost.

Ether is HelveX's fastest, lightest model. Streaming-first text and vision input, built for high-throughput workloads where every millisecond matters and the answer needs to land before the user has finished reading the question.

Capabilities

What Ether does well.

Four capabilities matter most when you're picking Ether on purpose.

Streaming-first

Server-sent events deliver tokens as they arrive. The user sees the reply forming in real time — no waiting for a complete response.

Vision input

Screenshots, diagrams, photos, PDFs as images — accepted at the same speed as text. Useful for support, moderation, document triage.

Function calling

Tool definitions are first-class. Ether routes between text generation and structured tool invocation deterministically when the schema says so.

Prompt caching

Long shared prefixes — system prompts, knowledge bundles, instructions — are cached. Identical follow-up calls run at a fraction of the cost.

Where it fits

Built for volume and latency, not depth.

Common shapes of work where Ether is the right pick. For depth-bound or action-bound work, route to Nexus or Prometheus.

  1. 01

    Real-time chat surfaces

    Customer-facing assistants where the user is watching the reply form. Voice replies, in-app helpers, support widgets, sales chat.

  2. 02

    Batch classification

    Tagging large volumes of tickets, transcripts, leads, or documents on a schedule. Stream-first means the queue drains fast.

  3. 03

    Support triage and routing

    First-line responses, intent detection, smart routing to the right human or the right deeper model. Ether handles the front door, Nexus handles the hard ones.

  4. 04

    Drafts, rewrites, and summaries

    Quick editorial passes — punching up an email, condensing a meeting note, rewriting a paragraph to a target tone.

  5. 05

    Vision triage

    Screenshots from a bug report, photos from a moderation queue, scans of a paper form. First-pass description and routing without waiting on a heavier model.

  6. 06

    Tool-call orchestration at speed

    Short agent loops where each step is a tool call and the planning is light. Ether keeps the loop fast; Prometheus takes over when the plan gets recursive.

Technical posture

How Ether behaves in production.

The operational posture you can plan around when you wire Ether into a real system.

  • Inputs Text and image. Single-prompt and multi-turn chat. Tool definitions accepted alongside content.
  • Output mode Streaming server-sent events by default. Tokens delivered as they're produced. Batch / non-streaming mode is opt-in.
  • Tool use Function calling with strict JSON schema enforcement. Parallel tool calls supported for independent tools.
  • Caching Prompt caching for repeated prefixes. System prompts and pinned context bundles cost a fraction after the first call.
  • Region Available in every HelveX region. On enterprise plans, runs inside your AWS or GCP tenant.
  • Best paired with A routing layer that escalates depth-bound questions to Nexus, and action-bound questions to Prometheus.
