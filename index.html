Workspace · Chat

Talk to the model that fits the moment.

A single thread that escalates from instant to deep to autonomous. Ether for speed, Nexus when accuracy matters, Prometheus when the work itself needs doing — same memory, same context, no copy-paste.

How it works

  1. Open a new chat or pick up where you left off — past threads stay in the sidebar tagged with the model that wrote them.
  2. Type, talk, or drop a file. Inline attachments up to 100 MB.
  3. HelveX picks the model based on your latency, depth, and budget targets — or you pin one yourself.
  4. Need more depth mid-thread? Press ⌘E to escalate. Full conversation context comes with you.
  5. Ready to act? Hand the thread to a Prometheus agent — it inherits everything said so far.

Capabilities

  • Mid-thread escalation (⌘E)
  • Multilingual responses
  • Image understanding
  • File attachments
  • Forked threads
  • Saved snippets
  • @-mentions
  • Diff view
  • Auto-pruning context
  • Resume from any message

Workflow examples

Drafting

Tighten a 2,000-word draft

Ether returns a tightened pass in 3 seconds. You spot a fact that needs verifying. ⌘E to Nexus — same thread, same draft, now with citations.

Translation

German support reply, formal register

Ether streams the translation in under a second, preserves the original tone, flags one idiom worth a human eye.

Transcript

Drop the audio. Ether transcribes, isolates the figures, returns a markdown table you can paste anywhere.

  • Forking experiments keeps the main thread intact.

Workspace · Projects

Folders that remember everything you put in.

Pin sources, instructions, defaults, and teammates once. Every chat inside the project starts from there. Context becomes a setting, not a ritual.

How it works

  1. Create a project for a body of work — a launch, a paper, a customer.
  2. Pin files (up to 1 GB combined), web links, and a custom system prompt.
  3. Choose a default model and tone. Set sharing — private, team, or Space-wide.
  4. Open any chat inside the project. Pinned context is preloaded automatically.
  5. The project keeps a rolling summary of recent activity, so you always know where you left off.

Workspace · Agents

Send work into the background.

Describe an outcome, set the rules, walk away. Prometheus plans, takes the steps, asks for input only when it has to — and leaves a trace you can replay step by step.

How it works

  1. Describe the outcome in plain language. ("Draft a weekly competitor brief by 9am Monday.")
  2. Pick which tools and sources the agent can touch.
  3. Set guardrails: time budget, money budget, scope, approval points.
  4. Run once or schedule. The agent generates a plan and grades it before executing.
  5. Watch the live trace. Edit the plan mid-run, pause, or hand off to a teammate.
  6. On finish: the result, the replayable trace, and a cost summary.

Capabilities

  • Plan-and-critique loop
  • Sub-agents (parallel branches)
  • Pause & resume
  • Live plan editing
  • Deterministic replay
  • Schedules (cron-style)
  • Approval gates
  • Tool budget caps
  • Memory across runs
  • Notify on finish

Workspace · Knowledge

Your team's library, asked in plain English.

Upload anything. Connect anything. Ask anything. Every answer cites the source — and clicking the citation opens the document at the exact paragraph, slide, or audio timestamp.

How it works

  1. Connect sources — drag in files, paste URLs, or link Google Drive, SharePoint, Notion, or Slack.
  2. HelveX indexes everything: text, tables, images, audio, scans (with OCR).
  3. Ask in natural language across the entire library, or scope to one folder.
  4. Get an answer with citations. Click any citation to jump to source — at the right paragraph, slide, or timestamp.
  5. The library stays fresh. Sources re-index when they change, and answers flag if context has shifted.

Capabilities

  • PDF · DOCX · MD · CSV
  • Slides & spreadsheets
  • Audio & video transcription
  • OCR for scans
  • Permission-aware retrieval
  • Hybrid search
  • Citation timestamps
  • "Why this answer"
  • Cross-language retrieval
  • Folder-level scope

Workspace · Spaces

Shared rooms where the team thinks together.

A Space holds chats, projects, agents, and knowledge that the team co-owns. Mention each other, comment on any line, hand off context with one click — without losing what was said before.

How it works

  1. Create a Space for a team or initiative — sales, security, the Q4 launch.
  2. Invite teammates and set roles (owner, editor, commenter, viewer).
  3. Share chats, projects, and agents into the Space — they become co-owned.
  4. Comment on any message or step. Mention people with @, link to a moment in one click.
  5. Hand off a thread — the receiver gets the full context plus your note on what to do next.
  6. Track activity in a real-time feed, filterable by person, project, or agent.

Team controls

  • Role-based permissions
  • Real-time presence
  • Threaded comments
  • @-mentions
  • Handoff with context
  • Activity feed
  • Per-Space audit log
  • Shared knowledge library
  • Per-Space usage
  • Guest access

10 questions · plain answers

Specific questions.
Specific answers.

Which model should I use?

If the answer needs to land in seconds, Ether. If it needs to be right and well-sourced, Nexus. If it needs to actually go and do the thing, Prometheus. You can also let HelveX choose for you — it picks the cheapest model that meets your quality bar, and tells you which one it picked.

Will my data be used to train the models?

Never. Anything you write or upload into HelveX is not used to improve our models. Enterprise workspaces add a zero-retention mode that deletes inputs immediately after the response is delivered, plus a signed DPA covering breach SLAs and deletion guarantees.

Where is my data stored?

When you deploy via AWS or Google Cloud, prompts and responses stay in the cloud region you select. Audit logs can export to your own SIEM via the standard cloud-provider tooling.

How do you keep AI reliable in production?

Prometheus plans before it acts, checks itself at every step, and remembers what matters. You can pause an agent, look inside it, edit the plan, and resume. Every run leaves a replayable trace, so you and your team always know what happened and why.

How long is a typical engagement?

A typical first engagement runs 6–10 weeks: 1–2 weeks of scoping, 4–6 weeks of build, 1–2 weeks of verification and handover. After that most clients move into a monthly cadence where the same senior engineer keeps the systems we shipped sharp — re-tuning prompts, refreshing integrations, auditing cost and performance every cycle. Multilingual support is included by default; voice or speech features are scoped separately when needed.

How does HelveX handle accuracy and hallucinations?

Three concrete answers, not one slogan. (1) Nexus is grounded on retrieval whenever sources are available — answers cite the line they came from, and you can click through. (2) When confidence is low, the model says so, and points at what's missing instead of inventing. (3) For agent runs, every step writes to a replayable trace, so a wrong answer becomes a question you can audit and fix.
