For Developers

A clean API.
Real SDKs. Honest docs.

Three lines of code to your first call. The same API our team uses internally — typed, streaming-first, and built for production.

app.ts · @helvex/sdk 
// Stream a response from Ether — sub-100ms first token
import { helvex } from "@helvex/sdk";

const stream = await helvex.complete({
  model: "ether-1",
  stream: true,
  messages: [
    { role: "system", content: "You are a precise editor." },
    { role: "user",  content: "Tighten this paragraph." },
  ],
});

for await (const chunk of stream) process.stdout.write(chunk.delta);

Questions

Common questions.
Honest answers.

How is HelveX different from other AI providers?

Three frontier models tuned for distinct kinds of work, a single API to route between them, and a platform built for production from day one — including agents that actually finish multi-hour tasks. We focus on the specific moments AI matters: speed when you need it (Ether), depth when you need it (Nexus), action when you need it (Prometheus).

Do you train on customer data?

Never. Inputs and outputs through the API and platform are not used for training. Enterprise plans include zero-retention mode and signed DPAs. Your data is yours.

Where is data processed?

Default residency is US-East. EU residency (Frankfurt + Zurich) is available on Pro and Enterprise. Choose at the workspace level. Cross-region routing is opt-in, never automatic.

How do agents stay reliable over long horizons?

Prometheus combines structured planning, plan critique, and a memory hierarchy with verifiable checkpoints. Agents can be paused, inspected, and resumed. Audit trails are first-class — every step is traceable and replayable.

Can we self-host or run in our VPC?

Yes — Enterprise plans support private deployments inside your AWS, GCP, or Azure account, with VPC peering and customer-managed keys. Talk to sales for the architecture review.

What happens if a model is down?

Multi-region failover is automatic. Our 99.99% SLA is contractual and credit-backed on Enterprise. Live status, incident postmortems, and a public changelog at status.helvex.ai.
