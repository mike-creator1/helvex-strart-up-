Principles

Seven things, in every shipped surface.

The principles that show up in every engagement that lands well. None of them are clever. All of them are operational.

  1. 01

    Pick the model on purpose

    Default routing is fine for most calls. The wins come from pinning Ether, Nexus, or Prometheus on purpose where the shape of the work is obvious.

  2. 02

    Use Projects for repeated work

    Anything you do twice belongs in a Project. System prompt pinned, knowledge seeded, prompt cache warm. Repeated work gets cheaper and faster.

  3. 03

    Treat citations as the answer

    A RAG response without a citation is a guess. Render citations inline, link back to the source, refuse confidently when nothing supports the question.

  4. 04

    Put humans on irreversible steps

    Sending email, posting comments, mutating CRM, writing to production — every one of these is a gate. Reversible reads stay automatic.

  5. 05

    Replay before you blame

    When the agent gets it wrong, replay the trace before you change the prompt. Most of the time the model isn't the problem; the retrieval or the tool wiring is.

  6. 06

    Write down what worked

    Successful prompts, retrieval strategies, eval set additions — all written down. Tribal knowledge evaporates; docs compound.

  7. 07

    Keep the model out of the user

    The user shouldn't have to think about which model is running. Routing is your job; the surface should feel like one assistant, even when it's three.

Anti-patterns

Five shapes we won't ship.

The patterns that show up in every engagement that stalls. Worth recognising before they take root.

  1. 01

    Treating the chat as the product

    A chat box wrapped around a model is the demo, not the surface. The product is the workflow around it.

  2. 02

    Skipping the model picker

    Defaulting every call to the same model wastes either cost or quality. Pick on purpose; pin where it matters.

  3. 03

    Building without citations

    An answer that can't be traced is a guess. Citations are how a retrieval system earns its keep.

  4. 04

    Skipping approval gates on irreversible writes

    A single bad agent action that goes out without a human gate undoes weeks of trust. The gate is cheap; the recovery isn't.

  5. 05

    No replay budget

    Without time set aside to replay failures, the same agent breaks the same way every month. Replay is the cheapest engineering work you do.

Operating cadence

One meeting a month — Workspace Refresh.

The single recurring meeting that keeps the workspace healthy. Four agenda items, ninety minutes, same time every month.

  • Cost review Last-month spend by surface and model. Outliers identified, routing decisions revisited where appropriate.
  • Failure review Three to five failed agent traces, replayed live. Root cause noted, runbook updated.
  • Roadmap pick The next workflow or surface to ship. One choice, written down before the meeting ends.
  • Doc update The runbook gets the changes from the last month merged in. Docs leave the meeting up to date.
