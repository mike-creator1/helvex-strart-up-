Where it fits

Common shapes of work where Prometheus is the right pick. For latency-bound or volume work, route to Ether or Nexus first.

  1. 01

    Agentic engagements

    Multi-step tasks where the agent decides what to do next. Inbox triage to action, research to summary, plan to execution.

  2. 02

    Complex analysis

    Investment memos, market scans, technical due diligence — work that reads a lot, synthesises across sources, and produces a defended position.

  3. 03

    Code at depth

    Cross-file refactors, architecture reviews, bug hunts that need to read the whole repo before suggesting a change.

  4. 04

    Computer-use automations

    Driving legacy apps that don't have an API. The agent runs in a sandboxed VM, with every step logged and replayable.

  5. 05

    Research with sources

    Web search + extended thinking + citations. The pattern for analyst-grade write-ups.

  6. 06

    Overnight runs

    Long-running engagements you start at end-of-day and review in the morning. Trace is fully replayable.

Technical posture

How Prometheus behaves in production.

The operational posture you can plan around when you wire Prometheus into a real system.

  • Thinking Extended (visible). The whole reasoning path is visible. Pause, edit a step, resume.
  • Tool use Parallel · chained. Independent tools run in parallel. Dependent tools chain. Plans branch and retry.
  • Computer use Screen · keyboard · mouse. Operates a sandboxed VM. Approval gates enforced for irreversible actions.
  • Web search Native tool. Search, fetch, read, cite — all inside the same call.
  • Replayability Every run leaves a step-by-step trace. Replay from any step with edits.
  • Best paired with Approval gates on writes / sends / payments. Ether as the front door, Nexus for retrieval-grounded analysis.
